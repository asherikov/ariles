6 Introduction
7 ============
8 
9 Ariles is a C++ serialization/configuration library with some extra automatic
10 code generation capabilities. It relies on third-party libraries for support of
11 various data representation formats. The library also provides some predefined
12 serialization wrappers for common types as described below.
13 
14 Ariles was not designed with an intention to address certain limitations of
15 other serialization libraries such as https://uscilab.github.io/cereal/ or
16 https://www.boost.org/doc/libs/1_67_0/libs/serialization/doc/, but rather
17 accidentally evolved from a bunch YAML serialization macro. Ariles, however,
18 differs from the common C++ serialization libs in several aspects:
19 
20 * It is intended to be used for both serialization and configuration,
21  therefore, it assumes that the input files could be written by a human or
22  generated by another program. For this reason, Ariles tries to be flexible
23  while parsing, e.g., for most formats it silently ignores unused entries,
24  ignores ordering of entries, does not discriminate attributes from childs in
25  XML, can optionally ignore missing entries, etc.
26 
27 * Ariles is meant to be used primarily for 'invasive' serialization, so it
28  generates class methods for serialization, initialization to default values,
29  and postprocessing after reading data from a file. Also, Ariles relies on
30  virtual methods for serialization via base classes (unlike boost and cereal).
31 
32 * Ariles supports some 'exotic' serialization formats, such as ROS parameter
33  server and Octave script.

36 Supported formats
37 =================
38 
39 Currently supported formats are (all are optional):
40 
41 * YAML via yaml-cpp, both old C++03 and new API supported.
42  - yaml-cpp does not comply with the specification when it emits NaN's and
43  infinities, see https://github.com/jbeder/yaml-cpp/issues/507. Ariles
44  includes a workaround for this issue.
45 
46 * msgpack via msgpack-c.
47 
48 * JSON via RapidJSON, with optional Jsonnet (https://jsonnet.org/)
49  preprocessing:
50  * NaN's and infinities, which are not allowed by JSON specification, are
51  parsed / saved anyway using `boost::lexical_cast`.
52 
53 * XML via PugiXML:
54  - Attributes are treated as childs while reading and are never used for
55  writing.
56 
57 * Octave script, output only, no dependencies:
58  - Ariles outputs a serialized class to an Octave script file, which can
59  later be `source`d from Octave to create a struct representation of the
60  class.
61 
62  - Eigen matrices are written in the 'native' format, so they can be used
63  directly, no reshaping is necessary.
64 
65  - Matlab might be supported, but has not been tested.
66 
67 * ROS parameter server, via standard ROS libs.

81 Minimal example
82 ===============
83 Class [`./tests/types/minimal.h`]:
84 ```
85 class Configurable : public ariles::ConfigurableBase
86 {
87  #define ARILES_SECTION_ID "ConfigurableEntryName"
88  #define ARILES_AUTO_DEFAULTS
89  #define ARILES_ENTRIES \
90  ARILES_TYPED_ENTRY(integer_member, int)
91  #include ARILES_INITIALIZE
92 };
93 ```
94 
95 Serialization:
96 ```
97 Configurable configurable;
98 configurable.writeConfig<ariles::yaml_cpp>("config_file.yaml");
99 ```
100 
101 Result:
102 ```
103 ConfigurableEntryName:
104  integer_member: 0
105 ```
106 
107 Deserialization:
108 ```
109 Configurable configurable;
110 configurable.readConfig<ariles::yaml_cpp>("config_file.yaml");
111 ```
