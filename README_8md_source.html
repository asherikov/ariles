1 Introduction
2 ============
3 
4 Ariles is a C++ serialization/configuration library with some extra automatic
5 code generation capabilities. It relies on third-party libraries for support of
6 various data representation formats. The library also provides some predefined
7 serialization wrappers for common types as described below.
8 
9 
10 Supported formats
11 =================
12 
13 Currently supported formats are (all are optional):
14 
15 * yaml via yaml-cpp, both old C++03 and new API supported;
16 * msgpack via msgpack-c;
17 * JSON via RapidJSON, with optional Jsonnet (https://jsonnet.org/) preprocessing;
18 * Octave script, output only, no dependencies;
19 * ROS parameter server, via standard ROS libs.

32 Features
33 ========
34 
35 * Ariles, unlike many C++ serialization libraries, primarily relies on
36  preprocessor (similar ideas are described at
37  http://cplusplus.bordoon.com/dark_side.html). This allows to
38  - use standard virtual method mechanism for serialization via base classes,
39  which is not possible when templated serialization methods are used (as,
40  e.g., in boost serialization);
41  - avoid code repetition and related mistakes, for example, the following code
42  declares an integer variable and marks it for serialization with a single
43  directive:
44 ```
45 class Configurable : public ariles::ConfigurableBase
46 {
47  #define ARILES_SECTION_ID "Configurable"
48  #define ARILES_ENTRIES \
49  ARILES_TYPED_ENTRY(integer_member, int)
50  #include ARILES_INITIALIZE
51 
52 
53  public:
54  virtual void setDefaults()
55  {
56  integer_member = 10;
57  }
58 };
59 ```
60 
61 * Ariles generates utility methods to reduce amount of code necessary for
62  serialization / deserialization, e.g., the following initializes a class from
63  a YAML file (explicit instantiation of an 'archive' class is not needed):
64 ```
65 Configurable configurable;
66 configurable.readConfig<ariles::yaml_cpp>("config_file.yaml");
67 ```

70 Notes
71 =====
72 
73 Octave
74 ------
75 
76 * Ariles outputs a serialized class to an Octave script file, which can later
77  be `source`d from Octave to create a struct representation of the class.
78 
79 * Eigen matrices are written in the 'native' format, so they can be used
80  directly, no reshaping is necessary.
81 
82 * Matlab might be supported, but has not been tested.
83 
84 
85 yaml-cpp
86 --------
87 
88 * yaml-cpp does not comply with the specification when it emits NaN's and
89  infinities, see https://github.com/jbeder/yaml-cpp/issues/507. Ariles
90  includes a workaround for this issue.
91 
92 
93 RapidJSON
94 ---------
95 
96 * NaN's and infinities are enabled with corresponding flags, which is not
97  allowed by JSON specification, see https://github.com/Tencent/rapidjson/issues/972.
